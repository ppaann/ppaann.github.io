Refine style and improve UX for an old website UI

The task to redesign the webpage UI of the product, which I named it "C". The requirements are:

1. Look and feel is modernized 2. Usability improve under the circumstance that the original workflow is not influenced and no extra training needed. 3. Use as less development effort as possible: no regression, minimum functional change and testing

Day1, First meeting, 2 hours:

I meet my customer -- the product owner the tool. In the meeting, he showed me the UI of the tool and how it works in brief. I asked explicitly what bothered him the most and what were expected. I received the source code of the website and setup the environment.

Day2, planning, mockup and coding

Identify the problems 1. The look and feel need to follow the new company standard. 2. The page header component is very crowd 3. Items are not standardized. 4. Layout is not responsive

Solutions:

1. Use a website as reference. I let him to choose his preferred styles from couple of websites which meet the company standard. He chose the SharePoint website as reference. 2. Clean the header area to make it feels less busy. I ask the usage of the components in the header, which of them are more important and frequently used. Then I use meaningful icons to replace text and group and hide the less important components into dropdown menu. 3. Standardize the UI. I decide to use Foundation (http://foundation.zurb.com/) as the UI framework because of its compact size and SASS support.

I decided to start the work from refactoring the header. First, I collected all the functional components from the header, then grouped and aligned them. I was using Balsamiq to manipulate the icons and text, it helped me think.

I started working on coding right after I setup the Foundation environment. Here is the result of the header part

I demo the UI to the PO. He is happy about the change. Together we fix couple of code so the application performs same as before.
