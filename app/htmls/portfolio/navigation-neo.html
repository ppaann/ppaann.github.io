Design Navigation

We started a new project of building a website which combines the functionalities and contents from multiple tools and websites. We had very limited foresight about which tools and websites would be included in the new project, and very limited knowledge about what the features and contents had to be included.
Therefore, the task is to provide a solution which must be flexible enough to adapt: unknown number of contents, unknown number of data depth. And at the same time, it needs to be easy to understand, navigate and informative. We selected the website called Nuage, which was the primary website of the main service, as the starting point.

The navigation of the origin site.

Problem

The navigation on the old website is original designed for handling data and actions for focused content and user group. All users had to take trainings before they can start using the website and before they can complete tasks. Even so, the website frequently cause confusions and many questions raised.

  • 1. The structure of the navigation did not give space for expanding. In the website, there will be more categories and deeper information hierarchy. The current navigation has only one level at its main navigation, which gives to heavy task to the in-page navigation.
  • 2. The navigation didn't show the relationships of the components. The future website need to have more categories which connected to each other in their own way.
  • 3. The style of the whole website is old fashioned. By today's standard, for a intranet, the background color was too heavy and there were too many colorful elements in the page.

Solution

I started designing and prototyping the navigation for the new project.

  • Concentrated on the top area. Put more navigation levels there.
  • Define an architecture that for all components can follow
  • Provide possibility to support deep hierarchy
  • The content of the navigation should be more informative and can be contextual
  • User clean style and only use meaningful colors

Process

Evaluate current website

I first work with our product owner asking what kind of future project it would be, who were users and what were expected. On base of the information I had received, I planed the direction of my design.

  • Users were well trained with existing website, so in order to reduce the training cost the structure should not change too much
  • The count of categories would be expanded up to 10, but detail of each categories is unknown
  • The depth in each categories is unknown
  • It is very important to know the name of the content the user is working on
  • The navigation should shows at least the parent-children relationships between the content
  • As a company internal website, it should follow the company style guideline and keep the interface clean and simple.

Information architecture

At this point, I had to guess the future architecture and assume an extreme use case, because no one in the project plan was not finished and no one can be absolute sure what would come out in the end.

Seek for guide

I did not have too much time, so I start searching for inspirations immediately. One advantage to design an intranet website was the website must follow a style guideline, therefore, there was no too much options. I selected two: Mega-menu and breadcrumb
